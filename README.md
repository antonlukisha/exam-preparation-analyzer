# Анализатор подготовки к экзамену
[![Python](https://img.shields.io/badge/Python-3.13-blue?style=for-the-badge&logo=python)](https://python.org)
[![uv](https://img.shields.io/badge/uv-0.7.13-violet?style=for-the-badge&logo=uv)](https://github.com/astral-sh/uv)
## Описание

Скрипт для анализа данных о подготовке студентов к экзаменам. Обрабатывает CSV-файлы с информацией о сне, учебе, настроении и тратах на кофе. Формирует отчет с медианными тратами на кофе по каждому студенту за весь период сессии.

## Установка

```shell
git clone git@github.com:antonlukisha/exam-preparation-analyzer.git
cd exam-preparation-analyzer

uv sync
```

## Использование

### Базовый запуск

```shell
# Анализ одного файла
uv run app --files file.csv --report median-coffee

# Анализ нескольких файлов
uv run app --files file1.csv file2.csv file3.csv --report median-coffee

# Подробный вывод (режим с логами)
uv run app --files file.csv --report median-coffee --verbose
```

### Параметры

| Параметр | Описание |
|----------|----------|
| `--files` | Список CSV-файлов для анализа (через пробел) |
| `--report` | Тип отчета (доступен: `median-coffee`) |
| `--verbose`, `-v` | Включить подробное логирование |

### Пример работы

```shell
uv run app --files data.csv --report median-coffee
```

### Конфигурация

Переименуйте файл примера переменных окружения:
```shell
cp .env.example .env
```
Откройте этот файл и заполните его:
```env
# Путь к директории с данными (по умолчанию: data)
DATA_PATH=data
# Уровень логирования: DEBUG, INFO, WARNING, ERROR (по умолчанию: INFO)
LOGGER_LEVEL=INFO
```

## Архитектура

Проект спроектирован с возможностью легкого добавления новых отчетов:

```
src/
├── app/
│   ├── cli.py              # Точка входа, обработка аргументов
│   ├── config.py           # Конфигурация проекта
│   ├── logging.py          # Настройка логирования
│   ├── models.py           # Внутренние модели
│   ├── processors.py       # Формирование отчётных данных (доступны: median-coffee)
│   ├── reporters.py        # Генераторы отчетов
│   └── scripts.py          # Вспомогательные скрипты (линтер)
│   ├── utils.py            # Утилиты (чтение файлов, валидация)
└── tests/
    ├── test_cli.py
    ├── test_reporters.py
    ├── test_utils.py
    └── test_integration.py
```

### Добавление нового отчета

1. Создайте класс-обработчик, унаследовав его от базового класса `DataProcessor`. Реализуйте метод `process()`, который принимает словарь с данными студентов и возвращает список обработанных записей в нужном формате. *(processors.py)*
2. Зарегистрируйте обработчик в словаре `PROCESSORS`. *(processors.py)*
3. Добавьте тип в `choices` параметра `--report` *(cli.py)*

## Формат данных

CSV-файлы должны содержать следующие колонки:

| Колонка | Тип | Описание                 |
|---------|-----|--------------------------|
| student | str | ФИО студента             |
| date | date | Дата записи |
| coffee_spent | int | Траты на кофе            |
| sleep_hours | float | Часов сна                |
| study_hours | int | Часов учебы              |
| mood | str | Настроение               |
| exam | str | Экзамен                  |

### Пример данных

```csv
student,date,coffee_spent,sleep_hours,study_hours,mood,exam
Иван Иванов,2026-01-01,480,4.0,13,устал,Математика
Алексей Смирнов,2026-01-01,500,4.0,14,зомби,Физика
```

## Примечания

- Архитектура позволяет легко добавлять новые отчеты (паттерн стратеджи)
- Для сборки и управления использовался uv (0.7.13)
- Код соответствует PEP 8, для корректировки использовались black, isort, ruff
- Типизация с mypy
- Тесты покрывают критический функционал
- Использованы только стандартные библиотеки + tabulate для вывода
- Поддержка нескольких файлов и конфигурации через .env
